# Setting Up Google API Credentials for Maileg

To use Maileg, you need to set up Google API credentials for both the Gmail API and Google Calendar API. Follow the steps below to obtain and configure these credentials:

## Step 1: Create a Project in the Google Cloud Console

1. Go to the [Google Cloud Console](https://console.cloud.google.com/).

2. Click on the project drop-down and create a new project.

3. Give your project a name, and click "Create."

## Step 2: Enable the Gmail and Google Calendar APIs

1. In the Google Cloud Console, navigate to the "APIs & Services" > "Library" page.

2. Search for "Gmail API" and "Google Calendar API," and enable both for your project.

## Step 3: Create OAuth 2.0 Credentials

1. In the Google Cloud Console, navigate to the "APIs & Services" > "Credentials" page.

2. Click the "Create Credentials" drop-down and select "OAuth client ID."

3. Choose "Web application" as the application type.

4. Give your OAuth client a name.

5. Add the following authorized redirect URIs for Maileg:
   - http://localhost:8080/auth/callback

6. Click "Create" to create your OAuth 2.0 client ID.

## Step 4: Download JSON Credentials

1. After creating the OAuth client ID, click the download icon to download the JSON file containing your credentials. This file is typically named `credentials.json`.

## Step 5: Place Credentials in the Maileg Directory

1. Move the downloaded `credentials.json` file to the Maileg project directory.

2. Change the `credentials.json` file name to one with your email in it 'your.email1234@gmail.com.json'

## Step 6: Run Maileg

You are now ready to run Maileg. Make sure the script can access the `credentials.json` file you downloaded, and follow the instructions provided in the script's README.md to complete the setup.

If you encounter any issues during this setup process, don't hesitate to reach out for assistance.
